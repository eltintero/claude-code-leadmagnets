The Document That Makes AI Useful for New Hires, Managers, and Cross-Team Work

"Who handles billing disputes?"

"Where does our customer data live?"

"Who do I talk to about updating the website?"

Every company has dozens of these questions floating around. New hires ask them constantly. Existing employees still Google Slack for the answers.

Today you're creating two documents that solve this permanently:

1. Team Directory — who's on the team, what they own, who to route questions to.

2. Tool Stack Inventory — what tools you use, what data lives where, who owns each one.

Together, these let Claude answer routing questions instantly: "Who handles X?" and "Where does Y data live?" No more Slack archaeology.

Exercise 1: Team Directory • 15 minutes

Open your AI Knowledge Base Project. New conversation. Paste:
I want you to interview me to create a Team Directory for my company.

Ask me one at a time about:
- Each department or team (name, purpose)
- Who leads each team
- Key roles and responsibilities within each team
- Who owns what decisions
- Who to contact for common requests (IT help, HR questions, client issues, billing, etc.)
- Any contractors or external partners and what they handle

After the interview, generate a structured markdown file I can save as team-directory.md

Exercise 2: Tool Stack Inventory • 15 minutes

Same Project, new conversation:
I want you to interview me to create a Tool Stack Inventory for my company.

Ask me one at a time about:
- Every software tool we use (by department/function)
- What each tool is used for
- What data lives in each tool
- Who owns/administers each tool
- How tools connect to each other (integrations)
- Any tools we're planning to add or replace

After the interview, generate a structured markdown file I can save as tool-stack.md

Upload both documents to your Project.
